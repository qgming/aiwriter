# 神笔写作 v0.9.5 | 更新体验与日志能力增强

## 中文介绍

欢迎来到 **神笔写作 v0.9.5**。

这个版本聚焦于两类高频体验的打磨：
一类是 **应用更新相关体验**，让用户可以在应用内直接查看 GitHub 云端最新更新日志，并获得更清晰一致的更新弹窗；
另一类是 **写作工作流稳定性修复**，补齐资料库引用在正文写作链路中的传递，并修复设定删除确认弹窗被遮挡的问题。

### 本次版本重点

- 应用版本升级到 `0.9.5`
- 在「关于我们 > 更多信息」中新增 **更新日志** 入口
- 支持在应用内读取 **GitHub Releases 最新云端日志**
- 为 GitHub API 限流场景增加 Atom feed 回退，提升日志获取成功率
- 优化「有更新」弹窗布局，和更新日志窗口统一视觉与信息结构
- 修复写作页顶部栏设定详情中删除确认框被遮挡的问题
- 修复资料库引用未完整传递到正文写作与重新生成链路的问题
- 修复最近合并代码引入的构建错误，确保发布链路可继续使用

### 用户可感知改进

- 现在可以直接在应用内查看 GitHub 上最新发布版本的更新说明
- 更新弹窗展示更完整，版本信息、更新摘要、下载进度与操作区更清晰
- 删除人物档案、世界观、词条等设定时，确认弹窗不会再被上层窗口遮挡
- 在写作中选择的资料库内容，会更稳定地进入正文写作上下文

### 技术改进摘要

- 新增主进程拉取最新 GitHub Release 日志的接口
- 扩展 preload 与 renderer 类型定义，支持更新日志能力
- 新增独立的更新日志弹窗组件
- 调整确认弹窗为更高层级的 Teleport 渲染
- 补齐正文写作、重新生成消息时的 `selectedReferences` 数据流
- 清理资料库 store 中未使用导入，并修复 Electron 文件对话框类型问题

### 更新建议

如果你已经安装了旧版本，建议升级到此版本，以获得更一致的更新体验和更稳定的写作流程。

## English Introduction

Welcome to **AIWriter v0.9.5**.

This release focuses on two practical areas:
**update experience improvements** and **workflow reliability fixes**. Users can now read the latest GitHub-hosted release notes directly inside the app, while the update dialog has been redesigned to match the new release notes window. On the writing side, this version also fixes reference material propagation and confirmation dialog layering issues.

### Highlights in v0.9.5

- Application version updated to `0.9.5`
- Added a new **Release Notes** entry under About > More Information
- The app can now load the **latest cloud release notes from GitHub Releases**
- Added Atom feed fallback when the GitHub API hits rate limits
- Refined the update dialog to match the release notes modal layout and structure
- Fixed the delete confirmation dialog being covered inside writing setting details
- Fixed selected reference materials not fully flowing into content writing and regeneration paths
- Fixed build issues introduced by recent merged code so the release pipeline remains usable

### User-facing improvements

- You can now view the latest published changelog from GitHub without leaving the app
- Update information, download progress, and actions are presented in a clearer layout
- Delete confirmations for settings are no longer visually blocked by parent dialogs
- Reference materials selected during writing are now more reliably included in writing context

### Technical summary

- Added a main-process interface for fetching the latest GitHub release notes
- Extended preload and renderer typings for the new release notes capability
- Added a dedicated release notes modal
- Moved the shared confirmation modal to a safer top-level Teleport layer
- Completed `selectedReferences` propagation for content writing and regeneration
- Removed an unused import in the reference library store and fixed Electron dialog typing issues
